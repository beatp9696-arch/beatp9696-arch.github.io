เขียน 14 ก.ค. 2026 หลังจบงาน: โหมดแอป + Apple Health import + design system + ยุบรวมเข้า repo เว็บ + OS tabbar บนเว็บ

**หลักการจัดลำดับ:** ความเสี่ยงข้อมูลสูญ > ความถูกต้องของข้อมูล > ประโยชน์ต่อการใช้จริงทุกวัน > ความสวย
ทุกข้อบอก **ทำอะไร / ทำไม / ทำยังไง / แรง (S=ชั่วโมง M=ครึ่งวัน L=วัน+) / ความเสี่ยง**

## 🔴 P0 — ทำก่อน ไม่ทำแล้วเจ็บ (data safety)

### 0.1 Restore ข้อมูลจากไฟล์ backup  ·  S
- **ทำอะไร:** More → มีปุ่ม "Export all data" แล้ว แต่ **ยังไม่มีปุ่ม Import กลับ** — ตอนนี้ backup ที่ดาวน์โหลดไปเอากลับเข้าแอปไม่ได้เลย
- **ทำไม:** backup ที่ restore ไม่ได้ = ไม่ใช่ backup; เปลี่ยนเครื่อง/ล้าง Safari = ข้อมูลหายถาวร
- **ทำยังไง:** `app-shell.js` → แถว "Restore from backup" → file input → parse JSON → merge/replace ทีละ key (`pp-os:*`) → ถามยืนยันก่อนทับ + โชว์ว่าจะทับกี่วัน/กี่รายการ
- **ความเสี่ยง:** ต้อง validate schema ก่อนเขียนทับ ไม่งั้น backup พังทำข้อมูลดีพัง — เขียน guard + preview ก่อน commit

### 0.2 ย้าย localStorage → IndexedDB + ขอ persistent storage  ·  M
- **ทำอะไร:** เก็บข้อมูลใน IndexedDB แทน localStorage + เรียก `navigator.storage.persist()`
- **ทำไม:** **Safari ลบข้อมูลของเว็บที่ไม่ได้เปิด 7 วัน (ITP)** — ถ้า PP ไม่ได้ติดตั้งเป็นแอปหน้าจอโฮม ข้อมูลสุขภาพ/เงินหายเงียบๆ ได้จริง; localStorage ยังมีเพดาน ~5MB (Apple Health หลายร้อยวันชนได้)
- **ทำยังไง:** `js/core/storage.js` เป็น facade อยู่แล้ว (load/save/remove) → เปลี่ยนไส้ในเป็น IndexedDB (idb-keyval แบบเขียนเอง ~40 บรรทัด) + migrate ครั้งเดียวจาก localStorage + ขอ persist ตอน boot
- **ความเสี่ยง:** API เป็น sync อยู่ (`load()` คืนค่าเลย) แต่ IndexedDB เป็น async → ต้อง preload ทั้ง store ตอน boot เข้า memory cache แล้ว write-behind (แก้ storage.js ที่เดียว แอปอื่นไม่ต้องแตะ)

### 0.3 Auto-backup เตือนความจำ  ·  S
- **ทำอะไร:** ถ้าไม่ได้ export เกิน 30 วัน → ขึ้น banner ในหน้า Me
- **ทำไม:** backup ที่ต้องจำเอง = ไม่มีวันได้ทำ
- **ทำยังไง:** เก็บ `os.lastExport` ตอนกด export → เช็คในหน้า Me

## 🟠 P1 — ทำให้ใช้ได้จริงข้ามเครื่อง (ตอนนี้ iPhone กับ Mac ไม่รู้จักกัน)

### 1.1 Sync ข้าม 2 เครื่อง  ·  L  ← **ข้อใหญ่สุดในลิสต์**
- **ทำอะไร:** ข้อมูลบน iPhone กับ Mac เป็นชุดเดียวกัน
- **ทำไม:** ตอนนี้ localStorage แยกต่ออุปกรณ์ — บันทึกน้ำหนักบนมือถือ พอเปิดคอมไม่เห็น = ใช้จริงไม่ได้ (นี่คือเหตุผลที่แอปยัง "เล่นๆ" อยู่)
- **3 ทางเลือก (เลือก 1):**
  | ทาง | ข้อดี | ข้อเสีย |
  |---|---|---|
  | **A. GitHub Gist ส่วนตัว** (แนะนำ) | ไม่ต้องมีเซิร์ฟเวอร์, ฟรี, ใช้ PAT ที่เก็บในเครื่อง, มี version history ให้ย้อนได้ | ต้องเก็บ token ในเครื่อง (scope gist อย่างเดียว), sync แบบ manual/ตอนเปิดแอป |
  | **B. Supabase free** (เหมือน MiniMart) | realtime, มี auth, โตต่อได้ | ต้องดูแลโปรเจกต์ + RLS, ข้อมูลออกนอกเครื่อง |
  | **C. iCloud Drive + ไฟล์ JSON** | อยู่ใน ecosystem Apple, ไม่มี token | ต้องเลือกไฟล์เอง (เว็บเขียนไฟล์กลับไม่ได้) = กึ่งอัตโนมัติ |
- **ทำยังไง (ทาง A):** More → "Sync" → ใส่ PAT (scope: gist) → บันทึกใน storage → ตอนเปิดแอป pull → ตอนแก้ push (debounce) → conflict = last-write-wins ต่อ key + เก็บ `updatedAt` ต่อ record
- **ความเสี่ยง:** conflict resolution ถ้าแก้สองเครื่องพร้อมกัน — เริ่มจาก per-day/per-entry merge (ไม่ทับทั้งก้อน)

### 1.2 Apple Health sync ที่ทำงานในแอปที่ติดตั้งแล้ว  ·  M
- **ทำอะไร:** ปัจจุบัน `?hk=` (Shortcut → Open URL) **ใช้ได้เฉพาะ Safari** เพราะ PWA ที่ติดตั้งมี storage แยกจาก Safari
- **ทำไม:** PP น่าจะใช้ผ่านไอคอนหน้าจอโฮม = ทางที่อัตโนมัติที่สุดใช้ไม่ได้จริง
- **ทำยังไง:** Shortcut เขียนไฟล์ `pp-health.json` ลง iCloud Drive → ในแอปกด "Sync" ครั้งเดียวแล้วเลือกไฟล์ (ทำแล้ว) → **เพิ่ม:** ปุ่ม "Paste JSON" (วางจากคลิปบอร์ด, `navigator.clipboard.readText()`) = 2 แตะ ไม่ต้องเปิด Files
- **ต่อยอด:** ถ้าทำข้อ 1.1 แล้ว ให้ Shortcut ยิงเข้า Gist ตรงๆ → แอปดึงเองอัตโนมัติ **= ซิงก์สุขภาพอัตโนมัติจริง**

## 🟡 P2 — Health ลึกขึ้น (ตอนนี้ตื้นกว่าที่ข้อมูลมีให้)

### 2.1 กราฟแนวโน้มยาว + น้ำหนัก  ·  M
- ตอนนี้ย้อนหลังแค่ 7 วัน ทั้งที่ import มา 400 วัน — **ข้อมูลมีแต่ไม่ได้ใช้**
- เพิ่ม: สลับช่วง 7 / 30 / 90 / 365 วัน, กราฟเส้นน้ำหนัก + เส้นเฉลี่ยเคลื่อนที่ 7 วัน (น้ำหนักรายวันมี noise เยอะ ดูเส้นดิบไม่มีประโยชน์), แนวโน้มการนอน/ก้าวเป็นรายสัปดาห์

### 2.2 Goals ตั้งเองได้  ·  S
- ตอนนี้เป้าฝังในโค้ด (steps 8000 / water 8 / ex 45 / sleep 8) → ย้ายไป Settings เก็บใน `health.goals`

### 2.3 ดึงตัวชี้วัดเพิ่มจาก Apple Health export  ·  M
- ที่ export มีแต่ยังไม่ได้อ่าน: **resting heart rate, HRV, VO2max, workouts (ชนิด+ระยะเวลา), mindful minutes, State of Mind (iOS 17+)**
- HRV + resting HR = วัตถุดิบจริงของ "Recovery" แบบ WHOOP (ตอนนี้ Recovery ใช้ mood ที่กดเอง = อ่อน)
- **ทำยังไง:** เพิ่ม TYPE_MAP ใน `apple-health.js` (โครง parser รองรับอยู่แล้ว) + สูตร recovery = f(sleep, HRV vs baseline 30 วัน, resting HR vs baseline)

## 🟡 P3 — Money ลึกขึ้น

### 3.1 งบต่อหมวด + เตือนตอนใกล้เต็ม  ·  M
- ตั้งเพดานต่อหมวด/เดือน → แถบในหน้า Money เปลี่ยนสีตอน >80% → หน้า Me เตือน
- **ทำไม:** บันทึกรายจ่ายเฉยๆ ไม่เปลี่ยนพฤติกรรม; เพดาน+เตือนคือส่วนที่เปลี่ยน

### 3.2 รายการประจำ (recurring)  ·  S
- เงินเดือน/ค่าสมาชิก → กดครั้งเดียว ลงให้ทุกเดือนอัตโนมัติ (ลดแรงเสียดทาน = ข้อมูลครบ)

### 3.3 แนวโน้มข้ามเดือน  ·  M
- กราฟรายจ่าย/savings rate 12 เดือน + เทียบเดือนนี้กับค่าเฉลี่ย 3 เดือน

### 3.4 นำเข้า CSV จากธนาคาร/บัตร  ·  M
- กรอกมือทุกรายการ = ไม่ยั่งยืน; รับ CSV → map คอลัมน์ → จับหมวดอัตโนมัติจากคำในโน้ต (เรียนรู้จากที่เคยจัดหมวดไว้)

## 🟢 P4 — เชื่อมแอปกับ Moatrices (จุดที่ไม่มีใครมีเหมือน)

### 4.1 แท็บ Watchlist / Portfolio (local-only)  ·  L
- **ทำอะไร:** 12 ตัวใน watchlist → ราคาปัจจุบัน + implied growth จาก Reverse DCF + ลิงก์ไปบทความ deep-dive ของตัวนั้นบนเว็บ
- **ทำไม:** นี่คือ "all-in-one" ของจริง — สุขภาพ/เงิน/ความคิดการลงทุน อยู่ในแอปเดียว
- **ข้อควรระวัง:** พอร์ตจริงห้ามขึ้น repo สาธารณะ → เก็บใน local storage ของเครื่องเท่านั้น (โค้ดสาธารณะ ข้อมูลส่วนตัว) และไม่ sync ขึ้น Gist สาธารณะ (Gist ต้องเป็น secret)
- **ราคา:** FMP API key ฝั่ง client = โผล่ในโค้ด → ใช้ key ฟรีแบบ read-only + rate limit ต่ำ หรือให้ PP กรอกราคาเอง (ตรงกับกติกา CLAUDE.md ที่ว่าราคาถาม PP เสมอ)

### 4.2 อ่านบทความ Moatrices แบบ offline ในแอป  ·  M
- ดึง `feed.xml` → แสดงเป็นลิสต์ในแอป → กดอ่านได้แม้ออฟไลน์ (cache หน้าไว้) + "อ่านค้างไว้ที่ไหน"
- ตอนนี้ More เปิดเว็บได้ แต่ **ต้องมีเน็ต** และไม่มี state ว่าอ่านอะไรไปแล้ว

### 4.3 เว็บ Moatrices เป็น PWA ด้วย  ·  M
- ตอนนี้เว็บ **ไม่มี service worker** → ติดตั้งไม่ได้ ออฟไลน์ไม่ได้ (แอปมี, เว็บไม่มี)
- ทำ manifest + SW ให้เว็บ → ติดตั้งไอคอนเดียวได้ทั้งเว็บ+แอป (แถบแท็บ OS ที่เพิ่งทำจะสมบูรณ์)

## 🔵 P5 — คุณภาพ/โครงสร้าง (ไม่เห็นด้วยตา แต่กันงานพังในอนาคต)

### 5.1 Smoke test ที่ commit ไว้ใน repo + CI  ·  M
- ตอนนี้ทุกการตรวจอยู่ในสคริปต์ชั่วคราวใน scratchpad (หายทุก session)
- ย้ายเป็น `pp-os/test/smoke.py` (CDP headless: เปิดทุกแท็บ, กดปุ่มจริง, เช็ค console error) → ต่อเข้า GitHub Actions ของเว็บ
- **ทำไม:** รอบนี้เจอเองสองบั๊กที่ตาไม่เห็น (ชีต Apple Health หลุดท้ายหน้า, หัวเว็บตกบรรทัด) — ครั้งหน้าไม่มีใครไล่ให้

### 5.2 หน้า "What's new" + เวอร์ชันในแอป  ·  S
- More → เลขเวอร์ชัน + สรุปที่เปลี่ยน (ตอนนี้ดูจาก sw VERSION อย่างเดียว)

### 5.3 a11y + Lighthouse pass  ·  S
- ตรวจ contrast จริง (โดยเฉพาะ `--dim` บนพื้นเข้ม), tap target ≥44px, aria ครบ

### 5.4 Web Push แจ้งเตือนรายวัน  ·  M
- iOS 16.4+ รองรับ web push **เฉพาะแอปที่ติดตั้งหน้าจอโฮม**
- เตือน "ยังไม่ได้บันทึกวันนี้" ตอน 3 ทุ่ม / "ซิงก์ Apple Health" ตอนเช้า
- **ต้องมีเซิร์ฟเวอร์ push** (หรือใช้ local notification trick ไม่ได้บนเว็บ) → ทำหลัง 1.1
