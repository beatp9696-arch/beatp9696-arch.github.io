ผ่าธุรกิจ MSFT (Microsoft) — 3 เครื่องยนต์, Azure และเดิมพัน AI

MSFT NASDAQ Microsoft Corporation
M
Moatrices
30 มิ.ย. 2026 · อ่าน ~15 นาที

Microsoft เป็นบริษัทที่ "น่าเบื่อ" ในแง่ดีที่สุด — รายได้ $281.7 พันล้าน กระจายอยู่บน 3 เครื่องยนต์ที่ต่างก็แข็งแรง ไม่มีตัวไหนล้มแล้วบริษัทพัง บทนี้อธิบายว่า มันหาเงินจากอะไร ทำไมองค์กรถึงย้ายหนียาก (คูเมือง) เดิมพัน AI/Copilot ไปถึงไหน และจุดที่ต้องจับตา

ตัวเลขอ้างอิงงบ FY2025 (สิ้นสุด 30 มิ.ย. 2025, 10-K) ควรตรวจสอบกับ filing ล่าสุดก่อนใช้ตัดสินใจ เป็นการวิเคราะห์เพื่อการศึกษา ไม่ใช่คำแนะนำซื้อขาย — เน้นอธิบายโครงสร้างธุรกิจมากกว่ามูลค่า/ราคาเป้า

1. ธุรกิจนี้หาเงินจากอะไร

จุดแข็งแรกของ Microsoft คือ มันไม่ได้พึ่งอะไรอันเดียว — รายงานเป็น 3 segment ที่ต่างก็เป็นธุรกิจระดับแสนล้าน:

  • Productivity & Business Processes ($120.8B, +13%) — Microsoft 365 (Office), Teams, LinkedIn, Dynamics
  • Intelligent Cloud ($106.3B, +21%) — Azure + server products (เครื่องยนต์โตหลัก)
  • More Personal Computing ($54.6B, +7%) — Windows, Xbox/Gaming (รวม Activision), Search

ความหลากหลายนี้คือ revenue durability — ต่อให้ขาใดขาหนึ่งสะดุด อีก 2 ขายังหมุนต่อได้

2. Azure — เครื่องยนต์ที่ตลาดจับตา

ถ้าให้เลือกเครื่องยนต์เดียวที่สำคัญต่ออนาคต คือ Azure — บริการคลาวด์ที่องค์กรเช่าใช้ compute/storage/AI แบบจ่ายตามการใช้งาน (consumption):

FY2025 Azure โต +34% และ Microsoft Cloud รวมแตะ $168.9B (+23%) โมเดล consumption มีข้อดีคือ ยิ่งลูกค้าใช้มากยิ่งจ่ายมาก — และคลื่น AI ยิ่งดันการใช้ compute บน Azure ขึ้นไปอีก (ลูกค้ารัน AI workload บน Azure OpenAI)

3. คูเมือง: ทำไมองค์กรย้ายหนียาก

คูเมืองที่แท้จริงของ Microsoft ไม่ใช่ผลิตภัณฑ์ตัวใดตัวหนึ่ง แต่คือ การที่ทุกอย่างผูกกัน ผ่าน identity และ data เดียว:

องค์กรที่ใช้ Windows + Microsoft 365 + Teams + Azure + Active Directory ฝังลึกแล้ว การจะย้ายค่าย หมายถึง retrain พนักงานทั้งบริษัท + ย้าย data + เสี่ยง downtime — switching cost ที่สูงจน CIO ส่วนใหญ่เลือกอยู่ต่อ นี่คือ Switching Costs + Scale Economies แบบ Helmer

4. Copilot — วาง AI ทับฐานลูกค้าเดิม

กลยุทธ์ AI ของ Microsoft ฉลาดตรงที่ไม่ได้เริ่มจากศูนย์ — มันเอา OpenAI/GPT มาวางเป็น ชั้น Copilot ทับผลิตภัณฑ์ที่องค์กรใช้อยู่แล้ว:

ผลคือ Microsoft ขาย AI เพิ่มบนฐานลูกค้าองค์กรเดิม (M365 Copilot, GitHub Copilot, Azure AI) โดยไม่ต้องหาลูกค้าใหม่ — เป็น distribution advantage ที่สตาร์ทอัพ AI ไม่มี

5. ตัวเลขบอกอะไร

  • รายได้ใหญ่และโตสม่ำเสมอ — $211.9B (2023) → $245.1B (2024) → $281.7B (2025, +15%)
  • Margin ระดับสูงมาก — operating income $128.5B = operating margin ~45.6%; net income $101.8B (~36%); diluted EPS $13.64
  • เครื่องจักรเงินสด + ลงทุน AI หนัก — cash from operations $136.2B (FY2025), อนุมัติ buyback ใหม่ $60B (ก.ย. 2024) — แต่ capex ก็พุ่งแรงเพื่อสร้าง datacenter รองรับ AI
ตัวเลขจากงบ FY2025 — โฟกัสที่ "ทำไมมันทนทานและคูเมืองอยู่ตรงไหน" มากกว่ามูลค่า/ราคาเป้า (บทนี้ไม่ทำ valuation)

6. เครื่องจักรเงินสด: Cash Flywheel

รายได้ส่วนใหญ่เป็น subscription + consumption ที่คาดการณ์ได้ → สร้างกระแสเงินสด มหาศาล → เอาไปลงทุน AI capex + R&D + ซื้อกิจการ (เช่น Activision) → ผลิตภัณฑ์แข็งขึ้นและ lock-in ลึกขึ้น → subscription เพิ่ม วงล้อนี้หมุนมาหลายปีและเป็นเหตุผลที่ Microsoft จ่ายปันผล + buyback ได้พร้อมกับลงทุนหนัก

7. Bull Case

  • คูเมือง switching cost + การกระจาย 3 ขา — รายได้ทนทาน ลูกค้าองค์กรเหนียว
  • Azure + AI ยังมี runway — องค์กรยังย้ายขึ้นคลาวด์ไม่หมด และ AI ดันการใช้ compute เพิ่ม Microsoft อยู่ในตำแหน่งเก็บเกี่ยวทั้งสองคลื่น
  • Distribution advantage ของ Copilot — ขาย AI บนฐานลูกค้าเดิมได้เลย + เครื่องจักรเงินสดที่ fund การลงทุนได้โดยไม่ต้องกู้เยอะ

8. Bear Case / จุดเปราะ

แม้ธุรกิจแกร่งมาก แต่มี 3 เรื่องที่ต้องจับตา:

  • AI capex มหาศาล — return ตามคาดไหม — การลงทุน datacenter หลักแสนล้าน ถ้ารายได้ AI ไม่ scale ทันหรือถูก commoditize ROIC จะน่าผิดหวังและกด margin
  • Azure อาจโตช้าลง — ฐานใหญ่ขึ้น + แข่งกับ AWS/Google ถ้า growth decel ชัด thesis ที่สองสะเทือน
  • Regulatory — antitrust เรื่อง bundling (เช่น Teams) และดีล M&A ใหญ่ (Activision) ถูกจับตาทั่วโลก อาจจำกัดการขยาย

9. Kill Conditions — อะไรจะทำให้ thesis พัง

วินัยสำคัญ: ต้องรู้ก่อนว่า "เหตุการณ์แบบไหนจะบอกว่าผมคิดผิด" สำหรับ MSFT คือ:

  • Azure growth ชะลอต่ำกว่า ~20% ติดกันหลายไตรมาส ขณะ capex ยังสูง → เดิมพัน AI capex ไม่ได้แปลงเป็นการเติบโต operating margin จะถูกกดจาก depreciation
  • Operating margin หดต่อเนื่อง จากต้นทุน AI/datacenter ที่รายได้ตามไม่ทัน → เศรษฐศาสตร์ของการลงทุน AI เริ่มไม่คุ้ม
  • Ecosystem lock-in เริ่มแตก — องค์กรย้ายออกจาก M365/Windows ไปคู่แข่ง (Google Workspace ฯลฯ) ในวงกว้าง หรือ regulatory บังคับแยก bundling → คูเมืองหลักเสื่อม

สรุป

MSFT เป็นตัวอย่างของธุรกิจคุณภาพที่คูเมืองมาจาก switching cost ขององค์กร + การกระจายรายได้ 3 ขา + เครื่องจักรเงินสด และกำลังต่อยอดด้วย AI/Copilot บนฐานลูกค้าเดิม จุดที่ต้องจับตาคือ เดิมพัน AI capex ที่หนักมาก ว่าจะให้ผลตอบแทนคุ้มไหม และ การแข่งขัน/กำกับดูแล ใครจะถือต้องตอบให้ได้ว่า ถ้า Azure โตช้าลงหรือ margin ถูกกดจาก capex จะคิดยังไง
